# Query Letter & Publishing Toolkit

**Purpose:** To generate publishing-ready assets from the completed memory bank. All the information needed for these documents already lives in the project's memory files — this guide shows how to assemble them.

Use after the manuscript is complete and revised.

---

## 📋 Back-Cover Blurb (150–200 words)

**Formula:** Hook → Character + World → Stakes → Question

**Template:**

```
[HOOK — 1 striking sentence that captures the emotional premise]

[SETUP — Who is the protagonist? What is their world? What do they want? 1-2 sentences]

[INCITING INCIDENT — What disrupts their world? What are the stakes? 1-2 sentences]

[STAKES QUESTION — End with an implicit or explicit question that the reader can only answer by reading]
```

**Data to pull from memory bank:**
- `Core/Synopsis.md` → Premise and inciting incident
- `Core/Characters.md` → Protagonist's core want and wound
- `Core/Story_Structure.md` → Central dramatic question

## 📄 1-Page Synopsis (500–700 words)

A synopsis tells the full story including the ending. Agents require this.

**Structure:**

1. **Opening** (1 paragraph): Protagonist, world, dominant desire, inciting incident
2. **Rising Action** (2–3 paragraphs): Key complications, escalating stakes, midpoint shift, key relationships
3. **All Is Lost** (1 paragraph): The dark moment and what the protagonist loses
4. **Climax & Resolution** (1 paragraph): What decision/action they take, how it resolves, and how the world/character has changed

**Tone rules:**
- Write in present tense, third person
- Focus on cause-and-effect, not scene description
- Do not use cliffhangers — reveal the ending
- Each character introduced must be named in ALL CAPS on first mention (e.g., **BAJIRAO**)

**Data to pull from memory bank:**
- `Core/Synopsis.md` → Full narrative arc
- `Core/Timeline.md` → Sequence of key events
- `Core/Characters.md` → Character arcs and relationships

## 📬 Query Letter (250–350 words)

**Structure:**

```
[Opening hook — 1 sentence that captures the book's emotional core or premise]

[Pitch paragraph — 2-3 sentences: protagonist, world, conflict, stakes. This is the back-cover blurb compressed]

[Book details — 1 sentence: Title, genre, word count, comp titles if applicable]
Example: "[TITLE] is a [WORD COUNT] [GENRE] in the tradition of [COMP TITLE 1] and [COMP TITLE 2]."

[Author bio — 1-2 sentences: relevant credentials, publications, or personal connection to subject]

[Closing — thank agent for their time, state what you're enclosing]
```

**Key rules:**
- Address the specific agent by name — never "Dear Agent"
- Do not pitch sequels in the query letter
- Do not describe the book as "a roller-coaster ride" or any other cliché

**Data to pull from memory bank:**
- `Core/projectbrief.md` → Genre, word count, and book concept
- `Core/Synopsis.md` → Pitch paragraph material

## 🔖 Logline (25–30 words)

A logline distills the entire book into one sentence. Used for pitches, social media, and personal clarity.

**Formula:** `[Protagonist with a flaw or want] must [pursue goal] before [stakes/deadline], or [consequence].`

**Example:** *"A disgraced Peshwa must forge an empire-spanning alliance before a rival coalition dismantles the Maratha confederacy — or lose everything he sacrificed his honor to build."*
